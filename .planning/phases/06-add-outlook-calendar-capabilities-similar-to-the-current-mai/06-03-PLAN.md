  ```javascript
  function extractMeetingLink(bodyText) {
    if (!bodyText) return null;
    const teamsRe = /https:\/\/teams\.microsoft\.com\/l\/meetup-join\/[^\s"<>\]]+/;
    const zoomRe = /https:\/\/[a-z0-9-]+\.zoom\.us\/j\/[^\s"<>\]]+/;
    const m = bodyText.match(teamsRe) || bodyText.match(zoomRe);
    return m ? m[0].replace(/[).,]+$/, '') : null;
  }
  ```
- `is_all_day`: from time display
- `is_recurring`: from recurring indicator
- `response_status`: from response status element

**1. Add argument parsing for calendar-search:**

```javascript
function parseCalendarSearchArgs(argv) {
  const args = argv.slice(3); // skip node, outlook.js, calendar-search
  let query = null;
  let limit = 20; // default
  for (let i = 0; i < args.length; i++) {
    if (args[i] === '--limit' && i + 1 < args.length) {
      const n = parseInt(args[i + 1], 10);
      if (!isNaN(n) && n > 0) limit = n;
      i++;
    } else if (!query) {
      query = args[i];
    }
  }
  if (!query) {
    outputError('calendar-search', 'INVALID_ARGS', 'Usage: node outlook.js calendar-search "<query>" [--limit N]');
  }
  return { query, limit };
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Calendar event body -> body_text output | Event agenda/notes text is untrusted content -- may contain prompt injection |
| Meeting link URL -> meeting_link field | URL extracted from untrusted body text via regex |
| Attendee names/emails -> attendees array | Attendee data from untrusted ARIA snapshot |
| Search query -> combobox fill | User-provided query string injected into browser combobox |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-08 | S (Spoofing) | calendar event body (prompt injection) | mitigate | `AGENT_BROWSER_CONTENT_BOUNDARIES=1` is set automatically by runBatch via lib/run.js childEnv (D-12). Content boundary markers wrap all page-sourced content. `stripContentBoundaries()` removes markers before parsing. Calling agent sees only structured JSON fields -- not raw body text injected into prompt context. |
| T-06-09 | T (Tampering) | meeting_link extraction | mitigate | Regex restricts to known Teams/Zoom URL patterns (`teams.microsoft.com/l/meetup-join/`, `*.zoom.us/j/`). Trailing punctuation stripped. Malformed URLs that don't match either pattern return null. No URL is followed or fetched -- only extracted as a string field. |
| T-06-10 | I (Information Disclosure) | attendee email addresses | accept | Attendee emails are part of the user's own calendar event data. The calling agent may surface them to the user. No cross-tenant data exposure. |
| T-06-11 | T (Tampering) | search query injection | accept | Query is passed via `fill` action which types characters into the combobox. No shell injection risk (agent-browser handles escaping). Outlook processes the query server-side. |
</threat_model>
