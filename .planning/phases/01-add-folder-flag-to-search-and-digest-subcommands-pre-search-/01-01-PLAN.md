   - If found, returns the alias value (the exact Outlook treeitem name)
   - If not found, returns `raw` unchanged (pass-through for custom folder names)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI argv -> batch command | `--folder` value from CLI is passed as `--name` argument to `agent-browser find` |
| agent-browser -> Outlook DOM | `find role treeitem click --name` performs substring match against ARIA tree |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | CLI argv | accept | Folder name comes from the calling agent (trusted). The CLI is not exposed to untrusted network input. Same trust model as existing --limit and query args. |
| T-01-02 | Tampering | batch command array | mitigate | Folder name is passed as a single array element to runBatch (JSON-serialized), not interpolated into a shell string. No shell injection vector exists — spawnSync receives args as an array. Verified: lib/run.js uses spawnSync with array args, not shell string. |
| T-01-03 | Information Disclosure | folder name in logs | accept | Folder name is logged to stderr (diagnostic only). Same pattern as existing query logging. stderr is not captured by calling agents. |
| T-01-04 | Elevation of Privilege | treeitem click | accept | `find role treeitem click` is a read-only navigation action. The action policy (policy-search.json, default: deny) already allows `click`. Clicking a folder treeitem does not grant write access — the action policy blocks all write operations regardless of which folder is active. |
| T-01-05 | Denial of Service | invalid folder name | accept | If `--folder "nonexistent"` is passed, `find role treeitem click --name "nonexistent"` will timeout and the batch will fail with a non-zero exit code. This is handled by the existing error path in executeComboboxSearch (returns null -> SESSION_INVALID error). Low severity — same failure mode as a network timeout. |
</threat_model>
