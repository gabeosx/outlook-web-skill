> These are not suggestions. Each constraint below was discovered through failure.
> Acknowledge each one explicitly before proceeding.

- [ ] CONSTRAINT: NO-PROFILE-FLAG — Never pass `--profile` pointing at an existing Chrome profile to agent-browser — it treats the path as a user data directory and corrupts the profile
- [ ] CONSTRAINT: FLAT-REFS-SNAPSHOT — agent-browser snapshot is a flat `{data: {refs: {eN: {role, name}}}}` dict, NOT a nested tree — `children`/`nodes` walking finds nothing

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| --profile with existing Chrome path | Passing `--profile ~/Library/.../Profile 1` to agent-browser treats it as a user data dir, creates `Profile 1/Default/`, corrupts the real profile. Wiped user's Chrome history and bookmarks. | blocking | Always use `--session-name outlook-skill` in run.js. Never use --profile with an existing Chrome directory. |
| snapshot children/nodes walk | agent-browser snapshot JSON is `{data: {refs: {eN: {role, name}}}}` — a flat dict keyed by element IDs. Walking `node.children` or `node.nodes` always returns nothing. Confirmed via diagnostic logging. | blocking | Rewrite extractMessageRows to iterate `Object.values(snapshot.data.refs)` looking for the target role/name. |
